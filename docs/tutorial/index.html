我的 Claude 使用教程

这是一页式的实践教程：先确认官方条件，再选择订阅或 API，最后按购买平台管理账单与问题。

最后实质核验：2026-07-10 · 作者：JumorHack

我把原先分散的使用经验重新整理成这条路径。它不是“保证开通”的捷径，而是为了让你先把产品边界、账户条件和费用责任弄明白，再选择下一步。

先说明边界：本站仅说明官方支持条件与公开的官方路径，不提供规避地区、身份、付款或账户限制的方法。产品规则与价格可能变化，最终以 Claude 官方页面和购买页面为准。

第 1 步：先确定你要解决的是什么问题

你的目标优先走的路径
在网页、桌面端或移动端使用 Claude先查看官方支持条件，再评估免费版、Pro 或当前可见的其他个人方案
把模型接入自己的程序、脚本或产品使用 Claude Console/API，并单独管理 API 账单、密钥和预算
想在终端里辅助编码先理解 Claude Code 的官方账户与认证方式，再选订阅或 API 路径

第一步决定了后面应查看的资料。最常见的误区是把聊天订阅当成 API 额度，或把 API 的按量费用当成聊天订阅的一部分。

第 2 步：确认你符合官方支持条件

在创建账户、升级方案或提交付款前，查看 Claude 的官方支持国家和地区。账户信息、账单资料和支付方式都应当真实、一致且可验证；不确定时直接使用官方支持入口确认。

第 3 步：如果是个人使用，先按官方订阅流程判断

  1. 先在官方支持地区内正常使用 Claude；
  2. 阅读Claude Pro 订阅指南，了解 Pro 的适用场景和不包含的内容；
  3. 在官方升级页或官方应用中核对所在地区实际显示的价格、税费与周期；
  4. 完成购买后，保留订单信息，并记下下次扣费日期。

需要比较 Free、Pro、Max、Team 和 API 时，直接看方案比较页，不要根据旧教程中的数字做决定。

第 4 步：遇到付款或账单问题，按归属渠道处理

付款被拒绝时，先核对资格、账单资料和发卡行提示，再联系相应的官方支持渠道；不要为了完成付款而变更身份或地区资料。可按付款被拒绝排查清单逐项确认。

需要停止续费或申请退款时，先确认订单是在网页、App Store 还是 Google Play 购买，再使用对应的官方管理入口。详情见取消与退款说明。

第 5 步：如果你要开发，不要跳过 API 账单设计

Claude Console/API 是独立的按量计费体系。开始开发前，先设置预算边界、区分开发与生产密钥，并了解模型、输入输出、缓存和工具如何影响成本。完整清单在Claude API 账单指南。

第 6 步：Claude Code 的选择

如果你在终端里使用 Claude Code，先判断自己是个人编码辅助，还是需要让应用自动调用模型。前者应关注订阅规则与额度，后者应关注 Console/API 的使用成本和密钥管理。可继续阅读Claude Code：订阅还是 API。
